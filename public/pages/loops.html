Mandatory 1 - Loops

for

for loops through a block of code a number of times. It is overly verbose, error prone since index can run out of bounds and it not functional

for (let i = let i = 0; i < 5; i++) { // code block to be executed }

forEach

forEach calls a function for each element in an array but it won't be executed for empty elements. forEach should be used when there are no need to return a new array

array.forEach(myFunction);

filter()

filter() creates a new array filled with elements that pass a test provided by a function. It does not execute the function for empty elements and it does not change the original array.

const drainedBatteries = carBatteries.filter(battery => battery < 0.5);

find()

find() returns the value of the first element that passes a test. It executes a function for each array element but not for empty elements. It will return undefined if no elements is found

const foundMovie = movies.find(movie => movie.id === Number(req.params.id));

findIndex()

findIndex() returns the index (position) of the first element that passes a test. It returns -1 if no match is found.

const foundMovieIndex = movies.findIndex(movie => movie.id === Number(req.params.id));
