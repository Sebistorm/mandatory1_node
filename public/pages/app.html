Mandatory 1 - App.js file

require

The require function is the easiest way to include modules that exist in separate files. The basic functionality of require is that it reads a JavaScript file, executes the file, and then proceeds to return the exports object.

const express = require("express"); const app = express();

listen

listen creates a server that listens on a chosen port of your computer.

const server = app.listen(8080, () => { console.log("The server is running on port", server.address().port); });

static

static serves static assets in Express which means that the client can get access to file type like css, js or html

app.use(express.static("public"));

readFileSync & toString

readFileSync is a programming interface of fs module which is used to read the file and return its content. toString makes it a string

const top = fs.readFileSync("./public/components/top.html").toString();

Body Parsing

express.json() is a built-in middleware function in Express. It parses incoming requests with JSON payloads and is based on body-parser

app.use(express.json());
